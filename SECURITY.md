# Security policy

Do not open public issues for credentials, private keys, tokens, cookies, personal data, confidential source documents, or undisclosed vulnerabilities. Use **Security → Report a vulnerability** in this GitHub repository; GitHub Private Vulnerability Reporting is the designated private channel. Do not include real client case material unless the maintainer has explicitly confirmed that it is necessary and authorized.

不得在公开 Issue 中提交凭据、私钥、令牌、Cookie、个人信息、保密来源文件或尚未披露的漏洞。请使用本仓库的 **Security → Report a vulnerability**；GitHub Private Vulnerability Reporting 是指定的私密报告渠道。除非维护者明确确认确有必要且已经授权，不得附带真实客户案件材料。

This repository must not be used to send case material to external services without a case-specific approval record.

未经案件级批准记录，不得利用本仓库把案件材料发送至外部服务。
